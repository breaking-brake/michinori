# DevOps × AI Agent Hackathon まとめ

> Google Cloud の AI 技術（Gemini Enterprise Agent Platform、Gemini 等）を必須技術とし、AI エージェントの **企画 → 開発 → デプロイ → 運用** までを一貫して実践するハッカソン。
> 主催: ファインディ株式会社／メインスポンサー: グーグル・クラウド・ジャパン合同会社

---

## 🎪 開催概要

- 生成AIの活用が「チャットで質問する」段階から「自律的に判断し実行するAIエージェント」へと進化する中、最前線の技術と開発プロセスを実際に手を動かして体験できる機会を提供する。
- 単なるプロトタイピングにとどまらず、実運用を見据えた **DevOps のプロセス全体** をカバーする。
- メインスポンサーにグーグル・クラウド・ジャパンを迎え、ツールスポンサー各社からの開発環境提供・技術アドバイスも受けられる。
- **2026/8/19（水）**、**Google 渋谷オフィス（渋谷ストリーム）** での最終ピッチに、審査で選ばれた **10チーム** を招待。
- **賞金総額 200万円**。経験者はもちろん、初参加・個人参加も歓迎。

### 🚩 3つのコンセプト

| コンセプト | 内容 |
|---|---|
| **つくる** | Google Cloud の AI を中核に、実務で役立つ独創的な AI エージェントを設計・実装する |
| **まわす** | GitHub 連携や CI/CD など DevOps のフローを構築し、AI を継続的に改善するサイクルを体験する |
| **とどける** | Cloud Run へのデプロイを通じ、スケーラブルな環境で本番品質のプロダクトをユーザーへ届ける |

### 🎓 学びをサポートする3つの仕掛け

- **🫱🏻‍🫲🏻 チームビルディングイベント** — チーム開発を経験したい個人参加者向けに 2026/6/7（日）開催。アイデアの発散からチームづくりまでをサポート。
- **⛺ Boot Camp** — グーグル・クラウド・ジャパン主催「Agentic AI Bootcamp 2026」＋各ツールベンダーの実践的オンラインコンテンツ（無料受講／事前申込制）。参加者全員のベースラインを引き上げる。
- **💡 審査員によるフィードバック** — AI エージェント開発で実績を持つエンジニアが審査員として参加し、提出作品に具体的なフィードバックを実施。9月のアフターイベント（オンライン）では「審査員の推し作品」も取り上げる。

### 🧑‍💻 こんな方におすすめ

## ⚙️ 開発要件

### 1. （必須）Google Cloud アプリケーション実行プロダクトの利用 ＊1つ以上

- App Engine・Google Compute Engine
- Google Kubernetes Engine (GKE)
- Cloud Run・Cloud Functions（旧 Cloud Functions）
- Cloud TPU / GPU

### 2. （必須）Google Cloud AI 技術の利用 ＊1つ以上

- **Gemini Enterprise Agent Platform**（旧 Vertex AI） — AI モデルのトレーニング・デプロイ・管理の統合プラットフォーム（AutoML、Vector Search、Explainable AI 等を含む）
- **Gemini API** — 最先端のマルチモーダル AI モデル。Gemini Enterprise Agent Platform 経由の利用を推奨するが直接利用も可
- Gemma・Imagen・Agent Builder
- **ADK (Agents Development Kit)**
- Speech-to-Text / Text-to-Speech API
- Vision AI API・Natural Language AI API
- Translation AI API

### 3. （任意）その他の技術の利用

- Flutter / Firebase / Veo / Elasticsearch（スポンサー）ほか、任意の技術を利用可能

## 💡 審査基準

1. **AIエージェントが価値の中心になっているか** — 単なる機能ではなく中核として機能しているか、自律的な振る舞い（判断・タスク実行）があるか、"AI エージェントである必然性" があるか。
2. **設定した課題へのアプローチ力** — 課題とその背景、対象ユーザー、提供価値といったプロダクトのストーリーに一貫性・妥当性・新規性があるか。
3. **ユーザビリティ** — ユーザーが直観的に使える機能やデザインを有しているか。
4. **実用性・体験価値の魅力** — 課題解決への実用性、効果的なアプローチができているか。突き抜けた体験価値は加点評価。
5. **実装力** — 技術・構成の納得度、拡張性、実運用への配慮など、必須ツールを活用したうえで総合的に評価。

## ⚙️ 応募方法

### STEP①：ハッカソンへの参加申し込み

- Findy Conference の申し込みフォームからエントリー。
- 個人・チーム参加とも **全員のお申込みが必要**。

### STEP②：作品をつくって Proto Pedia に登録

- 審査には作品共有プラットフォーム **「Proto Pedia」** を利用（事前にアカウント作成が必要）。

#### 【Proto Pedia へ作品を登録する際のルール】

| 入力項目 | 必須/任意 | 補足事項 |
|---|---|---|
| 作品ステータス | 必須 | 任意のステータスを選択。登録後の変更も可能 |
| 作品タイトル | 必須 | — |
| 概要 | 必須 | 作品の概要を記入 |
| 画像 | 任意 | 作品の紹介画像をアップロード可（最大5枚まで） |
| 動画 | 必須 | YouTube もしくは Vimeo にアップロードした動画の URL を入力 |
| システム構成 | 必須 | **「システムアーキテクチャ図」のアップロードを必須** とし、技術的補足があれば記入 |
| 開発素材 | 必須 | 開発に使用した開発ツールを記入 |
| タグ | 必須 | 複数設定可。1つ必須で **「findy_hackathon」** を設定 |
| ストーリー | 必須 | ①解決したい課題と背景 ②想定する利用ユーザー ③プロダクトの特徴 を記入 |
| メンバー登録 | 任意 | — |
| 関連URL | 任意 | — |

### STEP③：作品提出フォームから最終応募

- **作品提出フォーム（Google Form）** からの応募をもって正式エントリー受付が完了する。
- 応募にあたり、下記 3点 の提出が必要:
  1. **GitHub リポジトリの URL**（公開リポジトリ）
  2. **デプロイしたプロジェクトの URL**（動作確認できる状態にしておくこと）
  3. **「Proto Pedia」に登録した作品の URL**
- 詳細は作品提出フォーム内の項目を確認。

## ❓ Q&A

**Q. 年齢や職業に関わらず、だれでも参加できますか？**
A. 日本に居住する18歳以上の個人であれば参加可能（チーム参加も可）。ただし所属企業・団体の業務としては参加不可。国家公務員・地方公務員・独立行政法人の役職員・国立大学法人の役職員、その他法令により賞金等の受領に制限がある職務者は参加不可。

**Q. 参加申込み時に企業のメールアドレスの入力を求められますが、企業に所属しない個人も参加可能ですか？**
A. はい、可能。フリーランス・個人事業主は自身のメールアドレスを記入する。

**Q. チームビルディングイベント（6/7開催）への参加申込みはどこからできますか？**
A. ハッカソン参加申込み時に、申込フォームからチームビルディングイベントの申し込みが可能。

**Q. チームビルディングイベント（6/7開催）に参加すると、必ずチームを組まなくてはいけないのでしょうか？**
A. 必ずチームを組む必要はない。個人参加希望でも気軽に参加可。

**Q. 一人で複数の作品を提出することはできますか？**
A. はい、可能。その場合は作品ごとに提出する。

**Q. 一度作品を提出した後の再提出は可能ですか？**
A. はい、可能。最終応募時のタイムスタンプが最新の作品を正として審査する。

**Q. チームで参加する場合、全員が Proto Pedia のアカウントを作る必要がありますか？**
A. 強制はしない。代表者のみがアカウントを作成して投稿する形でも可。ただし最終応募の際にフォーム内でチームメンバー全員の氏名を記載する必要がある。

**Q. 応募作品の著作権はどうなりますか？**
A. 知的財産権（著作権法第27条・第28条の権利を含む）は参加者に帰属。ただし主催者は運営・広報活動に必要な掲載・転載などの権利を有する（詳細は「利用/参加規約」第10条を参照）。

**Q. SNS 発信をしたいのですが、ハッシュタグはありますか？**
A. 「**#findy_hackathon**」を利用。

**Q. 8/19（水）の決勝戦を会場で観覧することはできますか？**
A. はい、一般観覧が可能。参加申込み時に申込フォームから最終ピッチ（観覧）を申し込める。申込多数の場合は抽選。
