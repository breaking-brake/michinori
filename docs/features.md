# Michinori 機能仕様

## 概要

コードベースの現実から、タスクの全体像・依存関係・完了見込みを一気に可視化するプロジェクト管理ツール。

---

## DAG生成

GitHubパブリックリポジトリのURLとプロンプトを入力すると、AIがコードベースを解析しタスクDAGを自動生成する。

- **入力:** リポジトリURL + 自然言語プロンプト
- **処理:** git clone → ファイル解析（~80K文字予算） → Gemini 2.5 Flash（構造化JSON出力）
- **出力:** ノード（タスク名・工数MD・ステータス・カテゴリ・説明）+ エッジ（依存関係）

## AI相談（チャットハーネス）

DAG生成後、AIとチャットで対話的にDAGを修正できる。

- **POST /chat:** Gemini function calling (mode: AUTO)
- **propose_dag_changes ツール:** AIが自律的に「会話応答」か「DAG変更提案」かを判断
- **提案フロー:** 提案 → [DAGに反映/キャンセル] → プレビューダイアログ → [確定/閉じる]
- **プレビュー:** 変更後DAGをReact Flowで表示（追加=緑枠、変更=黄枠）
- **リポジトリ調査:** AIがソースコードを能動的に調査し、タスクの完了状況を判断可能
  - `list_files`: ファイル一覧取得（ディレクトリ絞り込み可）
  - `read_file`: 特定ファイルの内容読み取り
  - `search_code`: キーワードでコード全体を検索
  - 最大5ラウンドのfunction callingループで段階的調査
  - `/analyze`時のコードコンテキストをメモリキャッシュ（TTL 30分）

## クリティカルパス

- トポロジカルソート（Kahn's Algorithm）+ 最長経路DPで計算
- クリティカルパス上のエッジを赤くハイライト
- 完了予定日: 残りMD → 営業日ベース（土日・祝日スキップ）で算出
- 祝日データ: @holiday-jp/holiday_jp（日本の祝日対応）

## タスクカテゴリ

各タスクにカテゴリを割り当て、DAGノードにカテゴリ別色バッジを表示する。

| カテゴリ | 色 | 用途 |
|---------|------|------|
| 実装 | 紫 | コード変更（1PR単位を意識） |
| 調査 | シアン | リサーチ、PoC、ライブラリ評価 |
| 設計 | オレンジ | アーキテクチャ、API設計、スキーマ設計 |
| テスト | ピンク | テスト作成、QA、E2Eテスト |
| その他 | グレー | CI設定、ドキュメント、環境構築 |

カテゴリ・ステータスの定義は `packages/shared/src/schema/dag.ts` の `CATEGORY_DEFINITIONS` / `STATUS_DEFINITIONS` に一元管理。追加・変更は1箇所の編集で Zodスキーマ・Geminiプロンプト・UI色・ドロップダウン全てに反映。

## キャンバス操作

- **ノード追加:** 左上「+ ノード追加」ボタン → ビューポート中央に配置
- **ノード削除:** 選択時に右上❌ボタン（確認ダイアログ付き、接続も連動削除）
- **エッジ追加:** ハンドルからドラッグ接続（React Flow onConnect）
- **エッジ削除:** 選択時にベジェ中点❌ボタン（即削除）
- **ノードドラッグ:** 位置は永続化

## プロパティパネル

ノードクリックで右サイドパネルを開き、以下を編集可能:

- タイトル
- ステータス（ドロップダウン）
- カテゴリ（ドロップダウン）
- 説明（テキストエリア）
- 工数（MD、0.1刻み、onBlurで丸め補正）

## 工数

- 単位: MD（man-day, 8h = 1MD）
- 最小単位: 0.1MD
- ステータスによる残工数計算:
  - 未着手: estimateMd × 1.0
  - 進行中: estimateMd × 0.5
  - PR Open: 0
  - 完了: 0

## 稼働日カレンダー

完了予定日横の「稼働日設定」ボタンからカレンダーパネルを開き、稼働日をカスタマイズ可能。

- **プリセット:**
  - 平日稼働（デフォルト）: 平日 − 祝日 = 稼働日
  - 休日稼働: 土日祝 = 稼働日（個人開発用）
- **個別変更:** 日付クリックでプリセットの例外を設定（黄色枠表示）
- **過去日付は非活性**
- **永続化:** `.michinori.json` の `calendar` フィールド

## ファイル保存・読込

- **保存:** 現在のDAGを `.michinori.json` としてダウンロード
- **読込:** `.michinori.json` ファイルをアップロードしてDAGを復元（DAG未生成時も「読込」ボタン表示）
- **リセット:** 確認ダイアログ後、DAGとキャンバスを初期化

## レートリミット

Gemini API呼び出しのコスト保護。ミドルウェアで `/analyze` ルートのみに適用し、Gemini呼び出し前に拒否する。

- **IP単位:** 5回/分（`RATE_LIMIT_PER_IP` で変更可）
- **グローバル:** 30回/分（`RATE_LIMIT_GLOBAL` で変更可）
- **方式:** インメモリ・スライディングウィンドウ
- **超過時:** HTTP 429 + `Retry-After` ヘッダー
- **制約:** Cloud Runスケールtoゼロ再起動でカウンターリセット（ハッカソン用途では許容）

## 日次クォータ

サービス全体で1日あたりのリクエスト数を制限し、Gemini APIコストを管理する。

- **上限:** 100回/日（`DAILY_QUOTA_LIMIT` で変更可）
- **対象:** `/analyze` + `/chat` 合算
- **リセット:** JST 0:00
- **永続化:** Cloud Storage（`QUOTA_BUCKET`）に `daily-quota.json` を保存
  - コールドスタートでもカウンタが維持される
  - `QUOTA_BUCKET` 未設定時はインメモリフォールバック（開発時用）
  - バケット内ファイルは7日で自動削除（ライフサイクルルール）
- **開発者モード:** `NODE_ENV=development` で無制限
- **UIヘッダー:** 残回数表示（「リクエスト制限 残り N/100 回」）
- **超過時:** HTTP 429 + `Retry-After` ヘッダー

## Geoブロック（日本国内限定）

海外からのアクセスをアプリケーションレベルでブロックし、Gemini API課金リスクを低減する。

- **判定方式:** `geoip-lite`（MaxMind GeoLite2 DBバンドル、オフライン判定）
- **IP取得:** `X-Forwarded-For` → `CF-Connecting-IP` → fallback
- **ブロック対象:** 国コードがJP以外のIP
- **APIエンドポイント:** HTTP 403 + `{ error: "...", code: "GEO_BLOCKED" }`
- **トップページ:** 英語HTMLエラーページを表示
- **対象外:** `/health`（ヘルスチェック）、プライベート/ローカルIP
- **無効化:** `GEO_BLOCK_ENABLED=false` で完全スキップ
